# Review: plan-v3-restructure.md

## 一、架构方案评审：同仓库路由 vs 独立项目

### 结论：建议拆分为独立项目，而非同仓库路由

plan-v3 提出的"同一仓库 + 路由入口 + 两套流水线"方案看起来优雅，但实际执行存在以下问题：

**1. Agent "双模式"是伪复用**

planning-agent 在 dev 模式下做技术调研，在 research 模式下做背景调研——虽然都叫"调研"，但思路、工具使用方式、产出结构完全不同。把两套逻辑塞进一个 agent 文件里，只会让指令变得冗长且脆弱。修改一条线的逻辑时容易误伤另一条线。

**2. 根 CLAUDE.md 变成路由器后复杂度陡增**

现有的 CLAUDE.md 已经不短了，加上路由判断、两套阶段 A 适配表、类型传递逻辑，会变得很难维护。CLAUDE.md 的可读性直接影响 Lead 的执行质量。

**3. 调试困难**

如果 research 流水线出了问题，需要在共享的 agent 文件、workflows/research/、templates/research/、根 CLAUDE.md 四个地方来回排查。独立项目只需要看自己的文件。

**4. 真正共享的部分很少**

Git 策略、8 轮安全阀、检查点恢复——这些是机制性的"模式"，复制一份改改就行，不值得为此引入路由层的复杂度。

### 建议做法

复制当前项目到一个新目录（如 `Knowledge-Workflow`），在新目录里自由重构，不受现有代码开发流水线的约束。两个项目独立演进、独立使用。

## 二、Research 流水线设计评审

### 核心问题：流水线对场景的覆盖面不足

plan-v3 的 research 流水线按"调研报告"这一种场景设计（B1 背景调研 → B2 分析框架 → B3 产出物规划），但用户期望的场景远比这丰富。

以下是各场景与当前 B1→B2→B3 流程的匹配度分析：

| 场景 | 适合当前流程？ | 原因 |
|------|--------------|------|
| 深度调研报告 | 适合 | 流程就是为此设计的 |
| 参会指南 / 策略分析 | 基本适合 | 分析框架这步自然对应策略维度 |
| 评估方案 | 基本适合 | 分析框架对应评估维度 |
| 日程规划 | 不太适合 | 不需要"分析框架"，需要的是约束条件梳理 |
| 出题 / 出试卷 | 不适合 | 需要的是"知识点覆盖矩阵"而非"背景调研" |
| 批改考试 | 完全不适合 | 这是评判任务，不是产出任务 |
| 写作指南 / 教程 | 部分适合 | "分析框架"这步对教程类是多余的 |

### 改进建议

与其固定 B1/B2/B3 的含义，不如让阶段含义由需求驱动：

- **B1 永远是"主题调研/信息收集"** — 这是通用的，任何知识工作都需要先收集信息
- **B2 根据场景适配** — 可以是"分析框架"、"知识点矩阵"、"评估维度"、"约束条件梳理"等
- **B3 永远是"产出物规划"** — 这也是通用的，任何场景都需要明确交付物

具体的模板和检查清单可以准备多套，由 Lead 在阶段 A 结束时根据场景选择最匹配的组合。

## 三、术语补充：Knowledge Workflow

### 定义

用户所描述的工作流，在业界对应的术语是 **Knowledge Workflow**（知识工作流），或更完整地称为 **Knowledge Work Automation**（知识工作自动化）。

核心特征：**输入是需求/问题/素材，输出是结构化文本，中间需要调研、分析、组织、写作。** 边界判定——只要最终产出是"文档"而非"可运行的东西"，都属于此范畴。

### 相关术语

| 术语 | 侧重点 |
|------|--------|
| **Knowledge Work Automation** | 最贴切的总称，涵盖所有"输入需求，输出结构化文本"的场景 |
| **Content Production Pipeline** | 偏工程化的叫法，强调流水线式的生产过程 |
| **Research & Writing Workflow** | 侧重调研 + 写作的组合 |
| **Agentic Writing Pipeline** | AI Agent 领域的新兴叫法，强调多 Agent 协作产出文本 |

### 场景全景

以下是 Knowledge Workflow 覆盖的完整场景分类：

#### 用户已列出的场景

- 研究报告
- 指南
- 建议
- 评估方案
- 日程规划
- 深度调研
- 出题
- 出试卷
- 批改考试

#### 补充场景

**分析决策类**
- 竞品分析 / 市场调研报告
- 可行性评估（商业计划、投资标的）
- SWOT 分析 / 风险评估
- 政策影响分析

**教育培训类**
- 课程大纲 / 教学计划设计
- 学习路径规划
- 案例分析编写（商学院 case study 风格）
- 知识图谱梳理（某领域的概念关系整理）

**写作编辑类**
- 文献综述 / 读书笔记汇总
- 技术文档 / API 文档（非代码，纯文档）
- 白皮书
- 提案 / 标书 / 申请书撰写
- 翻译 + 本地化（不只是直译，需要调研目标文化语境）

**运营规划类**
- 会议纪要整理 + 行动项提取
- OKR / KPI 体系设计
- 流程 SOP 编写
- 项目复盘报告

**法律合规类**
- 合同条款分析 / 风险标注
- 合规检查报告
- 法律意见书草稿

**数据解读类**
- 数据分析报告（给定数据集，产出解读文档）
- 问卷设计 + 结果分析报告
- 年报 / 季报解读
